##### Software testing is a part of the quality assurance process. To automate this process we can use various tools. One of them is Selenium.

**Selenium** is an **open-source** software test automation framework. With the use of *Selenium WebDriver library*, it can be used to develop automation test cases for any browser. Each browser has its own dedicated driver. For example, you will need *ChromeDriver* for Chrome and *GeckoDriver* for Firefox. It's compatible with all the most popular languages such as Java, JavaScript, C, C#, Ruby and Python.

For the test case execution *Selenium WebDriver* relies on *JSON wire protocol*. We can say that *JSON wire protocol* works as a mediator between *client libraries* and the *WebDrivers*. It sends transfers data between the client and the server on the web.

Let's move on to the **advantages** and **disadvantages**. **Benefits** of using Selenium:

* Compatibility with different operating systems such as MacOS, Windows, Linux and Unix.
* Compatibility with all the most popular programming languages (Java, JavaScript, C, C#, Ruby and Python).
* Wide range of supported browsers such as Chrome, Firefox, Safari, and Edge.
* Open source with huge community.
* Pretty large library of plugins and extensions.
* Parallel test execution.
* It supports mobile testing (there are two main options – [Appium](http://appium.io/) and [Selendroid](http://selendroid.io/)).

**Disadvantages** of Selenium:

* Limited support for image testing.
* A time-consuming process for creating test scripts which requires programming knowledge.
* As it's open source - there is no reliable Tech Support available.
* There's no built-in reporting facility.
* The configuration of the test environment is very complex (compared to for example Cypress).

To **summarize** briefly. Selenium, despite its minor imperfections, is a powerful, widely used automation testing tool. It's definitely worth considering when choosing Automation Tool.
